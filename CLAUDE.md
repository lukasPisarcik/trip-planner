# Trip Planner

Personal trip planner built with SvelteKit + Bun + Tailwind 4 + shadcn-svelte. Every trip renders the same layout (Hero + tab bar: Itinerary, Transport, Viral Spots, Flights, Budget, Tips) — see `src/lib/components/trip/` for the reusable components and `src/lib/trips/` for trip data.

## Stack

- **Runtime:** Bun (`bun install`, `bun run <script>`, `bun add`, `bunx`). Prefer Bun APIs (`Bun.env`, `Bun.file`) over Node equivalents.
- **Framework:** SvelteKit 2 with Svelte 5 runes (`$state`, `$derived`, `$effect`).
- **Validation:** Zod (one schema file at `src/lib/schemas/schemas.ts`).
- **Logging:** Pino via `src/lib/helpers/logger.ts`.
- **Styling:** Tailwind 4 + shadcn-svelte primitives in `src/lib/components/ui/`.

## Hard rules (read before editing)

1. **All Zod schemas live in `src/lib/schemas/schemas.ts`.** Never define schemas inline or in feature folders. See `.claude/docs/schemas.md`.
2. **All env vars are declared in `src/lib/server/env.server.ts` (Zod-validated).** Never read `Bun.env` / `process.env` / `$env/*` directly elsewhere. See `.claude/docs/environment-variables.md`.
3. **Never edit files in `src/lib/components/ui/`.** Those are generated by shadcn-svelte; modify them via the CLI or by overriding in your own components.
4. **Server logic goes in `src/lib/server/services/`**, called from remote functions. See `.claude/docs/server-services.md`.
5. **Tests:** `*.test.ts` for server (Bun runtime, plain TS), `*.svelte.test.ts` for client (Vitest browser project, Svelte components). See `.claude/docs/testing.md`.

## Persistence

Trips and chat threads live in local JSON files at `.data/trips.json` and
`.data/chats.json` (gitignored). Trip data is strictly local and per-user:
on first read the trips file is seeded from any local seed modules in
`src/lib/trips/local/` (gitignored), so a fresh checkout starts with no
trips. All reads/writes go through `src/lib/server/data/store.ts` — see that
file for the file-store contract. Never read the JSON files directly from
feature code; go through the service layer (`src/lib/server/services/`).
